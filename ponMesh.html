Regenerative Programming

Applying Regenerative Programming to Build Automation Tools for Structural Analysis and Pre-Processing with HyperMesh

by Pon Rahul M

Problems faced in Finite Element Analysis

The problem with FEA is the time spent behind the pre-processing of the model. To perform FEA on any object, a large number of load-combinations and load-steps have to be defined in HyperMesh. This is a very time-consuming process for any analyst. When such analysis is done at an industrial level, it involves a large number of elements in a single body. Also, for performing correlation testing between FEM analysis and experimental results, engineers have to spend a considerable amount of time to organize and validate data. Generating load-combinations and load-steps manually caused many errors and was very time consuming as the number of combinations and cases are very large.

Regenerative Programming is the key

The task of entity creation can be automated by using TCL (Tool Control Language) script, which is the only script that is recognized by the Altair HyperWorks. The tool takes the inputs from an XLSX file containing the load case combinations in a pre-set format. By applying the principles of regenerative programming, TCL script can be generated. The TCL script when executed in HyperWorks generates the required load cases and load steps automatically.   Regenerative Programming is a programming methodology where a software/tool generates a block of code/script based on condition branching. The generated code need not be in the same language that was used for programming the generation software.

  Regenerative Programming has 3 phases namely
1.     Code Capture Phase
2.     Conditional Branching Phase
3.     Evolution Phase

a. Code Capture Phase:

   In the code capture phase, the TCL script generated for the different types of analysis are captured in a text file by generating each entity manually. This phase also includes the capturing of code generated while performing an attribute update.

b. Conditional Branching Phase:

     The variables and ID numbers are figured out in the generated script. These variables and IDs are replaced with generic strings with interpolated variables. Since our inputs are from an Excel sheet, an iterative process is carried out to generate a TCL script based on the selection made in the Excel sheet. Branching is based on the type of analysis and Pretension values.

   In the code capture phase, the TCL script generated for the different types of analysis are captured in a text file by generating each entity manually. This phase also includes the capturing of code generated while performing an attribute update.

c. Evolution Phase:

   Evolution or learning phase is carried out to increase the complexity of the code by increasing the number of analyses for which the TCL script can be generated. This is a very slow phase and a version update is issued when evolution is successful.
